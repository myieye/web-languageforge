Fields

In Language Forge, a field is a place where discreet a piece of lexical data or metadata is stored.

Field types

  • Some fields store data or metadata in one or more vernacular languages and others store it in one or more analysis writing systems. Each field displays the abbreviation of the language (input system) that the field is set up to use.

  • Some fields, such as the Word field, allow you to type in them. Other fields, like the Part of Speech field, store items from a list.

Fields that you can type in often have a X at the trailing end of the field. It clears the field of any contents so you can type new contents. Field that display list items typically show a drop-down list of items you can choose from.

Levels

In Dictionary views, fields store that data in three main levels:

  • Entry Level Fields

    • Word field stores the headword of the entry.

  • Meaning Level Fields

    • Meaning - This field stores a meaning of the headword.

    • Part of Speech - This field stores the part of speech for the corresponding meaning field. The parts of speech you can choose are controlled by a manager in the Option List tab.

    • Semantic Domain - This field stores the semantic domain for the corresponding meaning field.

  • Example Level Fields

    • Example - This field stores an example of the headword which corresponds to a particular meaning.

    • Translation - This field store the translation of the example in another language.
